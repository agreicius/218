Example 1.1.7. square roots: a non-example.

Explain why the following proposed rule does not define a function \(f\colon \R\rightarrow \R\text{:}\) given a real number \(x\in \R\text{,}\) define \(f(x)\) to be the square root of \(x\text{.}\)
🔗
Solution.
There are two issues with this proposed function.
  1. Firstly, not every real number \(x\) has a (real) square root! Consider \(x=-1\text{,}\) for example. We cannot have \(-1=y^2\) for a real number \(y\in \R\) since \(y^2\geq 0\) for any \(y\in \R\) and \(-1< 0\text{.}\) More generally, no negative real number has a square root.
    🔗
    🔗
  2. The specific issue described in (1) is easily remedied by replacing our domain \(\R\) with the smaller domain \(D\) of all nonnegative numbers \(x\geq 0\text{,}\) since it is a (nontrivial) fact that every nonnegative real number has a square root. However, a separate issue arises preventing \(f\) from being a well-defined function. Namely, with the exception of \(x=0\text{,}\) a nonnegative number has two distinct square roots. For example, the number \(x=4\) has two square roots: \(y=2\) and \(y=-2\text{.}\) This means there is no such thing as “the” square root, even if we restrict our attention to nonnegative numbers! Since our rule fails to assign a unique output for each input of the domain, it fails to be a function.
